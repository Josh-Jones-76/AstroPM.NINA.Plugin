1Build the night's time gridSessionScheduler.BuildTimeSlots

Establish the dark window and slice it into 5-minute slots. Each slot caches sun altitude, moon altitude, and moon illumination — every later decision reads from this grid.

  • scanStep from local 16:00 in 5-min steps ×216 (18 h)
    • civil dusk?sun crosses below −6° → record civilDusk
    • civil dawn?after dusk, sun crosses back above −6° → record civilDawn
  • found dusk?yes → window start = civilDusk − 1 h; no → fallback local 18:00
  • found dawn?yes → window end = civilDawn + 1 h; no → fallback start + 12 h
  • alignsnap start/end down to the 5-min grid
  • build slotsone TimeSlot per 5 min
    • cache SunAltDeg, MoonAltDeg, MoonIllumPct at slot start

For every target, decide which slots it can image, how moon-safe each slot is, classify its exposure sets into restrictiveness tiers, and total the remaining work. Targets that can't reach their minimum get dropped here.

    • slot dark?SunAlt < SunAltitudeThreshold
    • above min altitude?alt ≥ MinTargetAltitude

    • SlotUsable= dark AND above horizon
    • moon-OK ladder(project-level, for the V3-compat SlotMoonOk)
      • avoidance off?→ moon-ok
      • moonAlt ≤ MaxMoonAltitude?→ moon-ok (moon too low to matter)
      • moonIllum ≤ MaxIllumination?→ moon-ok (moon too dim)
      • moonSep ≥ requiredSep?RequiredMoonSeparation with width/relax falloff → ok / not-ok
  • longest usable run < MinTimeOnTarget?drop targetcontinue — never appears in the schedule this night

    • ES has a named profile?group by profile name; restrictiveness = sep × (1 + 100/(maxIllum+1))

3Build the schedule matrixScheduleEngine.BuildMatrix

Fold profiles + slots into the row × slot matrices the painter walks: who can image where, how moon-safe each cell is per tier, and per-target window metrics.

  • MoonDown[s]= moonAlt ≤ 0 (a night-wide flag)
  • per row → per slot
    • CanImage[r][s]= profile SlotUsable (altitude/dark only)
    • MoonSafe[r][s]= MoonDown OR profile SlotMoonOk
    • UsableSlot[r][s]tier-aware: true only if some tier with work can image here (moon-down safe for all)
    • track FirstUsable/LastUsable, MoonSafeSlots, MoonDownSlots, PeakAltitude
  • MinChunk sizing
    • base = MinTimeOnTargetHrs × 3600
    • total remaining < one min block?end-of-target completion: shrink MinChunk to the work that's actually left so the final frames survive the pre-filter
  • IsConstrained= usableSlots×300 < MinChunk×2 (tight window flag for sorting)
  • matrix windowFirstUsableSlot / LastUsableSlot / FirstDarkSlot / LastDarkSlot
4Overlap analysisScheduleEngine.ComputeOverlap

For each slot, count how many live targets could use it. Contention drives the exclusive-window reservation in the painter.

  • per slotcollect candidate rows
    • row pre-filtered?skip
    • CanImage & has work?add to candidate list
    • Contention= candidate count; IsExclusive when exactly 1
5Organize moon blocksScheduleEngine.OrganizeMoonBlocks

Pre-tag each unassigned slot with the tier of work it should host, so the painter spends scarce dark/low-moon time on the pickiest filters first.

  • slot is moon-down?
    • moon-downpick the most restrictive tier with work (No-Moon = ∞ wins)
    • moon-up → rising or setting?
      • moon risinguse relaxed tier first — save restrictive filters for later when they can't image
      • moon settinguse restrictive first — conditions improving, relaxed can wait
      • no safe tier → fall back to tier 0 (No LA)
6Paint the slots — strategy splitPaintSlots · PaintSlotsGreedy

The big allocation step: assign every imageable slot to a target. The chosen ImagingStrategy picks one of two painters.

  • strategy?
    • ManualPriority→ PaintSlotsGreedy (walk user priority order, give each target all it can use)
    • SharedTime (default)→ PaintSlots (multi-pass fair-share)
Both painters share the same shape: pre-filter → exclusive reserve → fill moon-down then moon-up → enforce minimums → prune slivers → bonus gap-fill → defragment → prune. They differ only in how the middle fill prioritises targets.

  • Pass 0exclusive windows — sole candidate slot is reserved immediately
  • Pass 0bextend a sub-minimum exclusive anchor outward until it reaches MinChunk
  • Pass 1 — moon-down LA
    • split candidatesexclusive (no moon-safe moon-up slots) vs flexible (can fall back to moon-up)

    • then fair-share flexibleon remaining moon-down slots

  • Pass 3 — moon-up

    • fair-share allocatorover moon-up slots, three branches:

      • minimums ≤ supplygive each its minimum, split excess proportionally to leftover demand
      • minimums > supply (SCARCE)greedily seat targets that fit their minimum; others get nothing this pass

6CShared painter sub-routinesEnforceMinimum · PruneSlivers · PaintChunks · DecrementWork

    • extendgrow the block into adjacent free imageable slots (after, then before)
    • borrowstill short → take edge slots from neighbours who stay above their own minimum
    • clearstill below minimum → release all of this target's slots (better idle than sub-min)
  • PruneSliversper target with a good (≥MinChunk) run plus shorter runs:

    • separated independent block?keep it (it's a real second visit) — unless it's a tiny scrap a neighbour fully covers

    • prefer runs adjacent to the target's own slots, then longest

  • DecrementWorkdebit hours by hint:

7Walk the painted matrix into a logWalkToLog → PickExposureSet

The execution simulation. Walk slot-by-slot through the painted assignments; inside each target block, run an exposure loop that advances a real clock. PickExposureSet is the per-exposure brain that chooses panel + filter.

  • emit Session start
  • per assigned slot
    • slot unassigned?start an idle timer; emit Wait if the gap ≥ 5 min
    • new target?
      • probe PickExposureSet — any work right now?
        • no workscan forward for the first viable slot; reassign only the leading dead slots to a fallback target that can image now
      • emit Slew, Center & Rotatereset filter cycle, dither counter, panel timers
    • exposure loop — while clock < slot end & night end
      • clock crossed into a slot owned by another target?break out of the block
      • multi-panel & too little time left to rotate?panel-lock to the current panel
      • PickExposureSetchoose ES (see 7A)
      • no pick & bonus enabled?
        • reuse last ES if still moon-safe, else re-pick allowing completed sets
        • still nothing?break
      • exposure would run past night end?end-of-night grace: allow exactly one final sub to overrun, then stop
      • panel changed?emit Slew→panel
      • filter changed?emit Filter
      • emit Image / BonusBonus when the ES has no remaining planned subs; advance clock; record exposure
      • dither enabled & subs-since ≥ ditherEvery?emit Dither
  • emit darkness begin/end & Session endtotals: minutes, subs, target count
7APickExposureSet — filter & panel decision treeSessionScheduler.PickExposureSet
  • build candidate set
    • panel allowed?honor panel-lock / mosaic restriction
    • exposure length > 0 & remaining > 0?(unless includeCompleted)
    • LA filter & not moon-safe here?rejectIsExposureSetMoonSafe
  • no candidates?return null
  • mosaic panel rotation
    • active panel has no LA work but another panel does (moon down)?forceOffCurrentPanel

  • candidate-pool selection
    • moon down & LA work available?keep LA candidates; honour current/next panel
    • moon up but some LA still safe?prefer non-LA on the active panel; rotate when due
    • active panel set?stay on it unless rotating
    • else → all candidates
  • moon-down & any "No Moon" profile filter present?exclusivity: those filters take the dark time alone until exhausted
  • urgency sort
    • primary: headroomleast remaining moon-safe seconds first (closing windows win)
    • tiebreak: hill directionmoon down/rising → most restrictive first; moon up/setting → least restrictive first
    • then: most start-of-night remaining workkeeps R/G/B balanced; def-order as final tiebreak
  • filter-switch cycling(when enabled)
    • under switch count & same filter still has work?continue current filter
    • at switch count — does any other filter fit its tolerance runway?
      • nostay on current filter (nothing else fits)
      • yesadvance to next filter in the tier loop that passes the runway check (wrap around)
        • time-critical guardcurrent window closes sooner AND lending would cost it work → hold current, but keep rotating within the equally-urgent group
  • return chosen ES+ panel label, panel index, ES index
8Validate the scheduleScheduleEngine.Validate

Post-walk sanity checks that surface warnings/errors in the simulator panel.

  • IDLE-GAP≥10 min idle while an imageable target was available (error)
  • LA-UNSAFEan LA filter imaged moon-up at an unsafe separation (error)
  • TOTAL-MINtarget got time but below its minimum block (warn)
  • NO-ALLOCtarget has work + usable time but got 0 subs (warn)
  • LA-MISStarget has LA work + moon-down slots but no moon-down allocation (warn)
